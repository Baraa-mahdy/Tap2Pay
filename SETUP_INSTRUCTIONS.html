üîí Firestore Security Rules

For Development (Test Mode): Already set automatically

For Production: Update Rules in Firestore Console:

rules_version = '2';
service cloud.firestore {
  match /databases/{database}/documents {
    match /suggestions/{document=**} {
      allow read: if request.auth != null && 
                     resource.data.status == "approved";
      allow create: if request.auth != null;
      allow update: if request.auth.token.email == "admin@tap2pay.com" && 
                       request.resource.data.status in ["approved", "rejected"];
    }
  }
}

üÜò Troubleshooting

Problem: "Firebase not configured" error

Solution: Check that you've added your Firebase config correctly in index.html. Open browser console (F12) and look for error messages.

Problem: Submissions not saving

Solution: Make sure:

  • Firestore Database is created
  • Rules allow writes (Test Mode does)
  • No JavaScript errors in console

Problem: Admin login not working

Solution:

  • Verify email is exactly: admin@tap2pay.com
  • Password must match what you set in Firebase
  • Check browser console for auth errors
